**기존 발표자가 `submissions/이름/SHOWCASE.md` (또는 폴더 내 다른 파일)를 수정**한 PR을 승인 처리합니다.
- 새 발표자(`submissions/이름/` 폴더 최초 추가) PR이면 `/pr승인` 을 안내하고 중단
- `slides/showcase.md` / `slides/showcase.html` 를 직접 편집한 PR이면 **레거시 패턴**이므로 경고하고 중단 (PR 작성자에게 `submissions/이름/SHOWCASE.md` 를 편집하도록 안내)

인자: `$ARGUMENTS` — 비어 있으면 `gh pr list` 로 열린 PR을 보여주고 사용자에게 확인받으세요.

## 아래 5단계를 순서대로 실행

### 1. PR 유효성 검사 + 머지
- `gh pr list` 로 열린 PR 확인
- 대상 PR의 변경 파일을 `gh pr view <번호> --json files -q '.files[].path'` 로 확인
- **검증 규칙**:
  - 변경 파일에 `submissions/이름/` 하위 파일이 포함되어 있어야 함 (이름은 기존 폴더)
  - `submissions/NEW이름/SHOWCASE.md` 가 **새로** 추가되었으면 중단하고 `/pr승인 <번호>` 안내
  - `slides/showcase.md` 또는 `slides/showcase.html` 만 변경하고 `submissions/` 변경이 없으면 **레거시 패턴**으로 중단
    - PR 작성자에게 "`submissions/이름/SHOWCASE.md` 를 편집하고 전체 빌드 실행 후 재제출" 안내
- 조건을 통과하면 `gh pr merge <번호> --merge` 로 머지 (squash 금지)

### 2. origin/main pull
```bash
git pull origin main
```

### 3. 변경 요약 표시
- `git log -1 --format="%an · %s"` 로 요청자명 + 머지 커밋 제목
- `git show --stat HEAD` 로 파일별 변경량
- 수정된 `submissions/이름/SHOWCASE.md` 의 의미 있는 텍스트 변경을 2~3줄 한국어 bullet 로 요약해서 사용자에게 보여주기

### 4. 전체 재빌드
- submissions 가 변경됐으므로 **반드시 전체 빌드** (옵션 없이)
```bash
bash scripts/build_slides.sh
```
- 스크립트가 `slides/showcase.md`, `slides/showcase.html`, `slides/showcase.pdf` 를 모두 갱신함

### 5. 커밋 & 푸시
- 재빌드로 변경된 파일: `slides/showcase.md`, `slides/showcase.html`, `slides/showcase.pdf` (+ 머지 커밋에 이미 포함된 `submissions/이름/` 은 중복 add 하지 않음)
- 한국어 커밋 메시지: `slides 재빌드 — <요청자명> 수정 반영`
- HEREDOC 으로 Co-Authored-By 포함하여 commit
- `git push origin main`
- 완료 후 GitHub Pages 링크 안내: https://arin-ship-it.github.io/lk-ai-camp2-biz-showcase/slides/showcase.html

## 실패/중단 조건
- PR 이 submissions/ 가 아닌 slides/ 만 수정 → 레거시 패턴 안내 후 중단
- PR 이 submissions/ 에 새 폴더를 추가 (최초 제출) → `/pr승인` 안내 후 중단
- merge conflict 발생 → 중단하고 사용자에게 상황 보고
- 빌드 실패 → 중단 (스크립트 stderr 그대로 전달)
- `git push` 실패 (예: non-fast-forward) → `--force` 쓰지 말고 중단 후 보고 (글로벌 CLAUDE.md 규칙)
