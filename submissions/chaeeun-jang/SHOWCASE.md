# Chaeeun Jang AI Camp2 Showcase

> 스크린샷, GIF, 스킬 파일 등 추가 파일이 있다면 같은 폴더에 함께 넣어주셔도 됩니다.

## ① 해결하고 싶은 문제

상시퍼널 성과 분석을 위해 매일 상시퍼널용 무료 클래스와 유료 클래스의 수강생 목록을 다운받아 엑셀 파일을 수동으로 분석하고 데이터를 기입하는 반복 작업이 병목이었습니다.
고객사에 성과를 보고할 때에도 클래스마다 1,000~2,000명 규모의 수강생 데이터를 받아 신청자 수, 완강률, 파트별 시청률, 결제 전환율 등을 직접 계산하고, 일자별·주차별·월별로 각각 정리한 뒤 노션에 옮기는 작업을 반복해야 했습니다.
상시퍼널을 적용하는 고객사가 늘어나면서 담당 클래스도 함께 늘어났고, 그에 비례해 이 작업에 드는 시간도 늘어났으며 실수가 생기기 쉬운 구조였습니다.

---

## ② 기존 방식의 병목

엑셀 파일을 받으면 매번 직접 함수를 작성해 집계해야 했기 때문에, 매일 반복하기가 현실적으로 어려웠고 일자별 데이터를 꾸준히 추적하기도 힘들었습니다.
파트별 이탈은 표로만 정리할 수 있었고 시각화가 불가능했습니다.
완성된 수치를 노션에 고객사가 이해하기 쉬운 형태로 옮기는 데도 시간이 많이 걸렸습니다.
유료 구매자 명단과 수강생 명단을 쿠폰코드와 수강생 인적사항으로 직접 매칭해 결제 전환율을 계산해야 했는데, 이 작업이 특히 오래 걸리고 오류가 많았습니다.

---

## ③ 지향했던 방향성

엑셀 파일을 폴더에 드롭하고 "inbox 처리해줘" 한 마디만 하면 분석부터 보고서 생성까지 전부 자동으로 처리되는 시스템을 만들고 싶었습니다.
결과물이 엑셀, 노션 DB, 노션 보고서 페이지, 구글 시트까지 한 번에 업데이트되어서, 숫자를 보고 인사이트를 도출하는 일에만 집중할 수 있는 구조를 목표로 했습니다.

---

## ④ 현재 구현 단계

전체 파이프라인이 동작하는 수준까지 완성됐습니다.

- **analyze.py**: VOD 수강생 엑셀을 읽어 일자별·주차별·월별·전체기간·파트별 시청률 5개 탭 결과 엑셀 자동 생성. 유료 구매자 명단이 있으면 이메일 매칭으로 결제 전환율·매출 자동 계산.
- **Notion 연동**: 분석 결과를 Notion DB 3개(일간 리포트, 인사이트 로그, 파트별 시청률)에 자동 기록하고, 파트별 바차트 시각화가 포함된 보고서 페이지도 자동 생성.
- **Google Sheets 연동**: 결과 데이터를 구글 시트 5개 탭에 누적 업데이트. 서식(헤더 색상, 교차 행, 증감 컬럼 빨강/파랑, 퍼센트 표시 등)까지 자동 적용.
- **실행 방법**: `inbox/` 폴더에 파일 드롭 → Claude Code에 "inbox 처리해줘" 입력 → 완료.

---

## ⑤ 추후 과제

- 여러 클래스 파일 동시 처리 시 안정성 검증 (10개 파일 동시 시나리오)
- 인사이트 자동 생성 고도화 (단순 수치 나열을 넘어 패턴 감지 및 액션 우선순위 자동 판단)
- 보고서를 고객사에 바로 공유할 수 있는 형태로 자동 포맷팅

## ⑥ 배운 점 / 소감

개발 경험이 전혀 없는 상태에서 실제로 동작하는 자동화 시스템을 만들 수 있다는 게 가장 큰 발견이었습니다.
수동 집계와 반복 작업에서 발생하던 실수와 시간 낭비가 줄어드니, 데이터를 보고 어떤 의미인지 해석하는 일에 더 집중할 수 있게 됐습니다.
세팅 과정에서 어려운 부분도 있었지만, 한번 완성하고 나니 업무 효율이 크게 높아졌고 고객사에 전달하는 보고서의 퀄리티도 눈에 띄게 올라갔습니다.
진작 시도해볼 걸 그랬다는 생각에 그동안 수작업하며 보낸 시간들이 스쳐 지나갔습니다. 앞으로 단순 반복 업무는 무조건 Claude에 맡기고, 나는 머리 쓰는 일에 집중하려고 합니다.
