## ① 해결하고 싶은 문제

<span class="hero-kw">자동화 · 자기복제 · 신속 학습</span>

- **자동화**: 통화 녹음 및 미팅 기록 자동화 / 신규 고객 온보딩 자동화
- **자기복제**: 세일즈 콘텐츠 기획(무료 강의안, 상세페이지, 광고소재, 모객 콘텐츠) 제작 에이전트
- **신속 학습**: 신규 강의 주제 도메인 이해도 높일 튜터

---

## ② 기존 방식의 병목

- 온보딩: 개별 온·오프 미팅 외 대안 없음
- 세일즈 기획: 매번 새로 기획·문서화·미팅 반복
- 기록: 매번 직접 기록
- 도메인 이해: 파편화된 정보 수동 리서치

---

## ③ 지향했던 방향성

- 온보딩: 고객사 LLM에 코드 입력 시 자동 구현 (부트캠프 Day 4 유사 구조)
- 세일즈 기획: 핵심 논리 방법론 + 자료 정교화 — 통합 스킬에서 개별 스킬로 분리해 효율화
- 기록: 녹음파일 자동 전달 — 현재 MacroDroid로 자동화 중
- 도메인 이해: 고속 학습 시스템 프롬프트 제작 + 고객사 강의 주제의 솔루션 페인포인트 타겟 확인 후 해당 내용 기반으로 학습 튜터 생성

---

## ④ 현재 구현 단계

<span class="hero-kw">역시 쉽지 않았다</span>

- 온보딩: 개별 온·오프 미팅 외 대안 없음 → 신규 온보딩 자동화: 미진행
- 세일즈 기획: 매번 새로 기획·문서화·미팅 반복 → 세일즈 기획 자동화: 무료 강의안 기획부터 일부 진행
- 기록: 매번 직접 기록 → 기록 자동화: 녹음파일 드라이브 업로드 구간 해결 중
- 도메인 이해: 파편화된 정보 수동 리서치 → 학습 튜터: 미진행
